# ðŸš— React Props Destructuring
When building components in React, you often pass data (called props) from parent to child components.

To make your code cleaner and more readable, destructuring is a great technique!

---

ðŸ“¦ What is Destructuring?

Destructuring lets you directly extract values from an object (like props) instead of accessing them one by one.

---

## âœ… Basic Destructuring in Function Parameters

If you know exactly which props your component needs, you can pull them out right in the function parameter list.

```jsx
function Car({ color }) {
  return <h2>My car is {color}!</h2>;
}

createRoot(document.getElementById('root')).render(
  <Car brand="Ford" model="Mustang" color="red" year={1969} />
);
```

Here, the component only cares about `color`, so we extract just that.

---

## ðŸ“¥ Destructuring Inside the Component
You can also destructure props after receiving the full `props` object.

```jsx
function Car(props) {
  const { brand, model } = props;
  return <h2>I love my {brand} {model}!</h2>;
}

createRoot(document.getElementById('root')).render(
  <Car brand="Ford" model="Mustang" color="red" year={1969} />
);
```
This way, you get access to the full `props` object, but still extract only what you need.

---

## ðŸŒŸ Using `...rest` to Catch Extra Props
If you want to get certain props and collect the remaining into one object, you can use the rest operator (...).

```jsx
function Car({ color, brand, ...rest }) {
  return <h2>My {brand} {rest.model} is {color}!</h2>;
}

createRoot(document.getElementById('root')).render(
  <Car brand="Ford" model="Mustang" color="red" year={1969} />
);
```
- Here, color and brand are extracted directly.
- rest will be: { model: "Mustang", year: 1969 }

---

## ðŸŽ¨ Setting Default Prop Values
If a prop might be missing, you can set a default value while destructuring.

```jsx
function Car({ color = "blue", brand }) {
  return <h2>My {color} {brand}!</h2>;
}

createRoot(document.getElementById('root')).render(
  <Car brand="Ford" />
);
```
Even though no `color` is passed, the default `"blue"` is used.
