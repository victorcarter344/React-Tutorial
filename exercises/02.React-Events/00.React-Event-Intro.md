## ðŸ“Œ What are React Events?
React events are synthetic events â€” they wrap around native browser events to provide a consistent and cross-browser interface.

Just like HTML, React supports events such as `click`, `change`, `submit`, etc. But in React:
- Event names use camelCase (e.g., `onClick` instead of `onclick`)
- Event handlers are passed as functions inside curly braces (e.g., `onClick={handleClick}`)

| Event          | Description                                  |
| -------------- | -------------------------------------------- |
| `onClick`      | Triggered when an element is clicked         |
| `onChange`     | Used for input fields when the value changes |
| `onSubmit`     | Fired when a form is submitted               |
| `onKeyDown`    | Fires when a key is pressed                  |
| `onKeyUp`      | Fires when a key is released                 |
| `onMouseEnter` | Triggered when the mouse enters an element   |

## ðŸ› ï¸ Handling Events in React

### âœ… Basic Event Handling

```jsx
import React, { Component } from 'react';

class App extends Component {
  state = {
    message: "Hello, welcome to React!",
  };

  handleClick = () => {
    this.setState({ message: "You clicked the button!" });
  };

  render() {
    return (
      <div>
        <h1>{this.state.message}</h1>
        <button onClick={this.handleClick}>Click Me</button>
      </div>
    );
  }
}

export default App;
```

## ðŸ“¥ Reading Props in Event Handlers
Props can be accessed inside event handlers, allowing dynamic behavior:
```jsx
function Button({ onClickHandler }) {
  return <button onClick={onClickHandler}>Click Me</button>;
}

function Parent() {
  const handleClick = () => {
    alert("Button clicked!");
  };

  return <Button onClickHandler={handleClick} />;
}
```

## ðŸ” Passing Handlers to Children
Child components can trigger functions defined in parent components by passing them as props.
```jsx

  };

    </div>
  );
```

## ðŸ”„ Event Propagation in React
React uses event bubbling â€” events move from child to parent unless stopped:

```jsx
function Child() {
  const handleClick = (e) => {
    e.stopPropagation(); // Prevents bubbling
    alert("Child clicked");
  };

  return <button onClick={handleClick}>Child Button</button>;
}

function Parent() {
  const handleClick = () => {
    alert("Parent clicked");
  };

  return (
    <div onClick={handleClick}>
      <Child />
    </div>
  );
}
```

## âŒ Preventing Default Behavior
Use `event.preventDefault()` to stop default browser actions like form submission.
```jsx
function MyForm() {
  const [name, setName] = React.useState("");

  const handleSubmit = (e) => {
    e.preventDefault();
    alert(`Name submitted: ${name}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input 
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Enter your name"
      />
      <button type="submit">Submit</button>
    </form>
  );
}
```

## ðŸ†š HTML DOM vs React DOM
| Feature         | HTML DOM                          | React DOM             |
| --------------- | --------------------------------- | --------------------- |
| Nature          | Real DOM                          | Virtual DOM           |
| Update Strategy | Direct DOM updates                | Virtual DOM diffing   |

| Event Handling  | Events are attached directly      | Uses synthetic events |
| Binding         | Manual binding                    | Uses state & props    |

## ðŸ–±ï¸ Final Example: Simple Controlled Input

```jsx
function UserForm() {
  const [name, setName] = React.useState("");
  const [submittedName, setSubmittedName] = React.useState("");

  const handleSubmit = () => {
    setSubmittedName(name);
    setName("");
  };

  return (
    <div>
      <input 
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Type your name"
      />
      <button onClick={handleSubmit}>Save</button>
      {submittedName && <h2>Hello, {submittedName}!</h2>}
    </div>
  );
}
```

## âœ… Summary

- React events follow camelCase (`onClick`, `onSubmit`)
- Handlers are passed as functions: `onClick={handleClick}`
- Use `event.preventDefault()` to override browser default
- Use props to pass handlers between components
- React uses synthetic events and event bubbling
